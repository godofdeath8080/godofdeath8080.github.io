android studio 2.0 preview 使用技巧

就在使用2.0preview的这段时间 发现谷歌已经默默的将priview 4升到9了 最新的相对稳定版是beta5 而且还支持自动更新
只能好评下谷歌程序员对IDE开发的热情

首次启动后如果卡住 将disable.android.first.run=true这句加到android studio
安装目录bin下面的idea.properties里

安装完成后快捷键改成eclipse的

在file-setting-keymap里的keymap选项改成eclispe

intellij本身定义好怪 比如ctrl+y是删除行 这个很不习惯 改成eclispe风格的算了

将SDK tools/platform tools都更新到 另外还有android support repository/android support
library/google repository这几个也安装上 可以避免在gradle时候报错

如果像体验as的高速虚拟机需要安装intel x86 haxm 实际其实还是慢 比xcode那个要慢多了 没有IOS爽
可能还是要依靠略残的genymotion

as还有个稍方便的地方 工程里想查看路径下的文件 eclispe只能右键properties把路径取出来在复制粘贴一串操作之后才能进入

现在右键show in explorer即可

添加库工程参见博客里的前一文 需要注意的是 库工程没法直接贴到as里 会提示cannot create class-file 还是需要文件目录拷贝

旧的adt下面的库工程可以用gradle export导出后 再在文件目录拷贝

另外库工程导入的编码设置遇到的一些问题 如果as默认设置是UTF-8 导入GBK工程 再改编码 然后rebuild 还是会编码报错

这时只能重启as了 不知道这算不算bug eclispe无此问题

显示行号 setting-editor-appearance-show line numbers

报错自动提示 alt+enter 但是好像包路径错误不会自动提示更正 ctrl+shift+o 也不会导正确的包 有时还是需要手动改下

as下修改工程名 实际是module名 直接rename会提示root module无法重命名 只有从文件系统重命名文件夹
然后将其中的*.iml改成新的工程名 重新打开即可

另外发现从eclipse直接导出的工程不会生成一个setting.gradle 如果有类似提示卡可以新建个工程然后拷到现有项目里

AndroidManifest里的targetsdkkversion 必须保持跟project structure里设置的编译版本一致
否则编译会报“Error:Execution failed for task ‘:processDebugManifest’”

GIT上面导出来的工程有的project级build.gradle导入报“Error:(19, 0) Gradle DSL method not
found: ‘android()’” 这种就把这段脚本删除 依赖如果非代码相关的也可以删除 只要module级的build.gradle不报错即可
eclipse里导出的工程只有一个build.gradle可以看成两个文件的结合
