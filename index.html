高山探秘｜南阿尔卑斯风光

无限解剖·旅行·2017 年 07 月 09 日

儿时一本《高山探秘》让我从小就对阿尔卑斯山区心驰神往。瑰丽的自然风光不是只有阿尔卑斯独享，丽江，稻城和世界上的很多山脉一样，都有着雪山，森林，湖泊这样的元素。但是阿尔卑斯并没有因为，成为风景区被过度保护和限制开发，站在山脚抬头仰望陡峭的山崖，站在山顶俯视幽深的河谷，总有几栋房子出现在你的视野里，几乎阿尔卑斯的每一个角落都能找到人类生活的痕迹，和城市中人们截然不同的生活方式，是阿尔卑斯山区最令我着迷的地方。

4月份和6月份去了两次南阿尔卑斯，途径了德国，奥地利，和意大利三个国家，4月份和老朋友一路从德国西面一路往南自驾，6月则是去见instagram上认识新朋友@leolinnnn老师。和大法A7一直携带的有三只镜头，28-70塑料挂机头，10-18 f4和28 f2，机器有点老了，不过我平时拍的主题器材没什么大要求，方便携带就好了。

慕尼黑的国王湖应该不少朋友去过了，旅游团标配。元首以前的老巢就在附近。

国王湖里的船都是电力驱动，建造年份跨度非常大，有些已经服役了几十年，有些则是近两年才下水。

游船的第一站，St. Bartholomew教堂。

obersee的网红屋。上湖每年的开放时间有限，需要乘船到最后一站后继续徒步。

新天鹅堡附近的小教堂。

第一次见原谅色的大众T3。

新天鹅堡，老天要吃人。远远看一下就好，这种景点绝对不买票进去。

路上经过的木屋。

冰洞山下的城堡。

夏天花花草草比较多。

山区里有不少充电桩，开特斯拉跑山也没有什么问题。

Messner Mountain Museum Corones，Zaha Hadid老师的作品，不过我妈说长得比较像垃圾中转站。

这张没舍得加滤镜，就搞了下颜色。

Lago di Braies 也算是网红景点了，直接背靠着山，是阿尔卑斯比较好看的一个湖。

星空P的，天天阴天，别提了。

Lago di Dobbiaco，不知为啥6月份的时候水比4月份脏很多。

Lago di Misurina旁边的疗养院，前往三峰石 3 Zinnen一定会经过这个湖。

被导航惯坏了，都不知道地图和指南针怎么用了吧。

傻牛。

三峰石下停车场遇到的西班牙情侣Anita和Joe，驾驶着他们的T3环游欧洲。能把旅行过成生活，着实令人羡慕。

到访三峰石可以一路开到最顶端的停车场，然后再徒步，一部车收费25欧元，房车40，算是阿尔卑斯除了滑雪之外收费比较高的景点。

开始徒步之后不久就会经过这个小教堂。

再继续往前就会到达寂静岭。

云里雾里还是很漂亮的。一般人这里就到顶了，进阶级登山家可以继续往上，穿过山洞之后有峭壁上带钢索的路。

车辆无法到达的地方，一样有酒店。

存在的目的就是免得登山客饿死在山上。

路虎卫士这个车虽然比较破，但是看到它的第一眼我就想炸死它。

能在欧洲强行过美国人生活也是服帖的。

这部T3和前面的两部不太一样，Syncro出品的T3带四驱。

阿尔卑斯山区住宅多全木结构，带着淡淡的松木香味。

奥地利的Hallstatt，严重过誉，如果前往尽量避免在镇上过夜。

教堂。

四月份那次还去了Antohlzer Lake，野火是允许的，但是一定注意防火，离开的时候要彻底熄灭。

能够离开自己平日熟悉的环境，背起背包前往未知的感觉真的很好。

自己的文字功力太差，所以一直希望能用照片来讲故事。从第一次接触数字尾巴到现在已经很多年了，不少人都在给数字尾巴打上各种标签。其实作为一个兴趣分享社区，能在这里结交到兴趣相投的新朋友就已经是足够了。

java.sql.SQLException:ResultSet.next was not called

中间层里sql查询报错java.sql.SQLException:ResultSet.next was not called

executeQuery返回的ResultSet对象必须使用while (rs.next())循环来获取每个子对象的列，直接取第一列就会报这个错。

executeUpdate返回的0或者1对结果没有任何影响，直接执行插入方法即可。

android studio中kotlin安卓工程的gradle配置

首先说说百度出来的教程的问题
我的环境用的AS3.0金丝雀版
kotlin extention for android这个插件被废弃了 不用再下了 kotlin插件本身就集成了
项目新建的时候可以选是否支持kotlin

创建成功后要在module的gradle里必须加上
apply plugin: ‘kotlin-android’
apply plugin: ‘kotlin-android-extensions’//这个有点蛋痛的 明明废弃了还要加上
另外
compile “org.jetbrains.kotlin:kotlin-stdlib-jre7:$kotlin_version”这里的kotlin_version在整个项目的gradle里要有
即ext.kotlin_version = ‘1.1.2-4’
配置完成后import kotlinx.android.synthetic.main.*就可以玩不写findViewbyId这些啦

kotlin的语法也还是比较简单的 java转过来还行
就是老项目有点坑爹 我试了下AS的转换 貌似没效果 不知道是不是AS版本的原因 只有一个个文件重构一遍

使用正式签名打包APK，打开debug模式报错的问题解决方法

SDK Tools Revision 8以后，如果在AndroidManifest.xml中直接设置打开debug模式：

1


android:debuggable="true"

则在使用正式签名打包APK时，Eclipse会报错如下：

Avoid hardcoding the debug mode; leaving it out allows debug and release
builds to automatically assign one

Issue: Checks for hardcoded values of Android :debuggable in the manifest
Id: HardcodedDebugMode

It’s best to leave out the android:debuggable attribute from the manifest. If
you do, then the tools will automatically insert android:debuggable=true when
building an APK to debug on an emulator or device. And when you perform a
release build, such as Exporting APK, it will automatically set it to false.

If on the other hand you specify a specific value in the manifest file, then
the tools will always use it. This can lead to accidentally publishing your
app with debug information.

解决方法如下：

将eclipse报错等级降低即可。

Eclipse -> Preferences -> Android -> Lint Error Checking -> Issue -> Security
-> HardCodeDebugMode -> Severity: -> Warning即可。如下图。

相关链接： http://developer.android.com/tools/sdk/tools-notes.html# SDK Tools Revision
8

Support for a true debug build. Developers no longer need to add the
android:debuggable attribute to the tag in the manifest — the build tools add
the attribute automatically. In Eclipse/ADT, all incremental builds are
assumed to be debug builds, so the tools insert android:debuggable=”true”.
When exporting a signed release build, the tools do not add the attribute. In
Ant, a ant debug command automatically inserts the android:debuggable=”true”
attribute, while ant release does not. If android:debuggable=”true” is
manually set, then ant release will actually do a debug build, rather than a
release build.

android studio 2.0 preview 使用技巧

就在使用2.0preview的这段时间 发现谷歌已经默默的将priview 4升到9了 最新的相对稳定版是beta5 而且还支持自动更新
只能好评下谷歌程序员对IDE开发的热情

首次启动后如果卡住 将disable.android.first.run=true这句加到android studio
安装目录bin下面的idea.properties里

安装完成后快捷键改成eclipse的

在file-setting-keymap里的keymap选项改成eclispe

intellij本身定义好怪 比如ctrl+y是删除行 这个很不习惯 改成eclispe风格的算了

将SDK tools/platform tools都更新到 另外还有android support repository/android support
library/google repository这几个也安装上 可以避免在gradle时候报错

如果像体验as的高速虚拟机需要安装intel x86 haxm 实际其实还是慢 比xcode那个要慢多了 没有IOS爽
可能还是要依靠略残的genymotion

as还有个稍方便的地方 工程里想查看路径下的文件 eclispe只能右键properties把路径取出来在复制粘贴一串操作之后才能进入

现在右键show in explorer即可

添加库工程参见博客里的前一文 需要注意的是 库工程没法直接贴到as里 会提示cannot create class-file 还是需要文件目录拷贝

旧的adt下面的库工程可以用gradle export导出后 再在文件目录拷贝

另外库工程导入的编码设置遇到的一些问题 如果as默认设置是UTF-8 导入GBK工程 再改编码 然后rebuild 还是会编码报错

这时只能重启as了 不知道这算不算bug eclispe无此问题

显示行号 setting-editor-appearance-show line numbers

报错自动提示 alt+enter 但是好像包路径错误不会自动提示更正 ctrl+shift+o 也不会导正确的包 有时还是需要手动改下

as下修改工程名 实际是module名 直接rename会提示root module无法重命名 只有从文件系统重命名文件夹
然后将其中的*.iml改成新的工程名 重新打开即可

另外发现从eclipse直接导出的工程不会生成一个setting.gradle 如果有类似提示卡可以新建个工程然后拷到现有项目里

AndroidManifest里的targetsdkkversion 必须保持跟project structure里设置的编译版本一致
否则编译会报“Error:Execution failed for task ‘:processDebugManifest’”

GIT上面导出来的工程有的project级build.gradle导入报“Error:(19, 0) Gradle DSL method not
found: ‘android()’” 这种就把这段脚本删除 依赖如果非代码相关的也可以删除 只要module级的build.gradle不报错即可
eclipse里导出的工程只有一个build.gradle可以看成两个文件的结合

Android Studio中导入第三方库工程的方法

http://blog.csdn.net/ocean20/article/details/44241629 原帖地址

1.下载所需库工程，此处以httpzoid为例子，在工程根目录下创建一个extras文件夹用于存放库工程。将httpzoid放到extras目录中

2.项目上点击右键打开Open Module Settings，点击红框中的加号添加一个module

选择导入库工程

按向导一步一步导入即可

导入module后，在modules下会多一个httpzoid

然后点击app的dependencies

点击红框中的加号然后在弹出对话框中选择刚才添加的module即可

gradle的相关配置文件会自动更新

例如：app中的

dependencies {
compile fileTree(dir: ‘libs’, include: [‘*.jar’])
apt “org.androidannotations:androidannotations:$AAVersion”
compile “org.androidannotations:androidannotations-api:$AAVersion”
// 编译extras目录下的Httpzoid模块
compile project(‘:Httpzoid’)
}

Project中的

include ‘:app’, ‘:Httpzoid’
project(‘:Httpzoid’).projectDir = new File(‘extras/Httpzoid’)

XX-net部署技巧

goagent3.2.3+gogotester废弃了有一段时间了

前段时间说gws ip都被封了

最近流行XX-net 于是试了下 部署挺简单

最新测试版 https://codeload.github.com/XX-net/XX-Net/zip/2.5.1

配置页面上传APPID 可以同时上传多个 扫描IP之后导入即可

chrome插件没用swichy 而是falcon proxy

现在chome扩展crx文件安装老是报错 从扩展地址末尾获取扩展程序ID

然后按此地址拼接
http://clients2.google.com/service/update2/crx?response=redirect&x=id%3D
扩展程序的ID %26uc%26lang%3Den-US&prod=chrome

再贴到浏览器下载 完成后到chome设置扩展页面拖进去就可安装

ADT通过XX-NET 更新

在\XX-Net-2.5.1\gae_proxy\local目录下找到proxy.ini 最后添加



[profile]


dl.google.com = nocrlf,noforcehttps,nofakehttps
